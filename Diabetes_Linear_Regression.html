Overview¶

Background¶

In this lab, we are given a dataset that describes a patients age, sex, bmi, blood pressure, and their blood. Using this information about many patients, we will try to create a model that predict the disease progression of diabetes.

Problem¶

This is a classic regression problem, where we are trying to predict a continuous value.

Methods¶

In this lab, we will be using Single Linear Regression and Multiple Linear Regression to find which method performs the best. Additionally, we will introduce some feature selection to see if we can get better results.

Data¶

This dataset can be found here. The diabetes dataset has 10 features, age, sex, bmi, bp, s1, s2, s3, s4, s5, s6. Using these 10 features and 442 observations, the goal is to predict the response variable, Disease Progression (of diabetes).

Randomized Feature Selection¶

Select random features 10,000 times and track the best RMSE and the features that correspond to that RMSE score.

Why?¶

In class, you discuss forward selection and backward selection, but this seems like a naive approach to me. Some features may not important without the presence of other features. Using forward selection will not capture those complexities.

Instead, I implemented randomized feature selection that randomly selects up to all of the features and tests the models performance. From here, I can pick the best combination of features.

In [175]:
import statsmodels.api as sm
import random
import statsmodels.api as sm

# Perform random feature selection and run 10000 times.
## Pick the best set of features.

best_r = 0 # Maximize this
best_rmse = 1000 # Minimize this
best_features_rmse = []
best_features_r = []

for i in range(10000):
    # Determine how many random features to generate.
    number_features = random.randint(1,len(df_features.columns))
    
    # Generate random features.
    feature_idx = []
    while len(feature_idx) < number_features:
        # Generate a random index.
        idx = random.randint(0, len(df_features.columns)-1)
        # Append index to feature selection if not used already.
        if idx not in feature_idx:
            feature_idx.append(idx)
    
    # Create df with just the features in the feature index.
    df_sub_features = df_features.iloc[:, feature_idx]
    
    # Split the data into training/testing sets
    diabetes_X_train = df_sub_features[:-20]
    diabetes_X_test = df_sub_features[-20:]

    # Create linear regression object
    lm = linear_model.LinearRegression()

    # Train the model using the training sets
    fit = lm.fit(diabetes_X_train, diabetes_y_train)

    # Make predictions using the testing set
    diabetes_y_pred = fit.predict(diabetes_X_test)

    # https://bigdata-madesimple.com/how-to-run-linear-regression-in-python-scikit-learn/
    # get feature names
    colnames = np.asarray(df_features.columns)

    #  scipy.stats OLS linear regression model
    X_train = sm.add_constant(diabetes_X_train)
    lm2 = sm.OLS(diabetes_y_train, X_train).fit()
    lm2.summary()
    
    rmse = np.sqrt( mean_squared_error(diabetes_y_test, diabetes_y_pred))
    r = r2_score(diabetes_y_test, diabetes_y_pred)
    
    # See if this is the best set of features we've seen so far.
    if best_rmse > rmse:
        best_rmse = rmse
        best_features_rmse = feature_idx
        
    if r > best_r:
        best_r = r
        best_features_r = feature_idx

# Print out the features and scores.
print(best_rmse, best_features_rmse)
print(best_r, best_features_r)

Model Recap¶

Single Linear Regression¶

rmse: 50.48

This model used bmi as an input variable to predict the Disease Progression Rate. With a Root Mean Squared Error of 50.48, this model performed the worst of the three models built. This model didn't perform well because we excluded some useful pieces of information that this model would find useful.

Multiple Linear Regression¶

rmse: 44.77

This model used All Features as input variables to predict the Disease Progression Rate. With a Root Mean Squared Error of 44.77, this model performed better than model 1, but worse than model 3. This model performed better than model 1 because it takes more features into account. However, this model performed worst than model 3 because of colinearity and noise. Some features are highly correlated with others (such as s1 and s2). Other features gave noise to the data set - these features were random, and provided little insight into predicting the response variable.

Mulitple Linear Regression (Alternative)¶

rmse: 42.90

This model used s5, sex, bmi and s4 as input variables to predict the Disease Progression Rate. With a Root Mean Squared Error of 42.90, this model performed the best. This model performed better than both model 1 and model 2 because it found the perfect combination of features to best explain the response variable.

This was done by implementing Randomized Feature Selection. This randomly samples features uses them to build a model. The model performance is calculated using RMSE, and then best set of randomly sampled features is recorded and used as our final features.

This was implemented because we need to find the combination of features that best describe the data. This may also be achieved by trying every combination of features, but I wanted to test this method out because sometimes that process would take too long. Random sampling can significantly reduce the time need to test samples, while still ensuring a good amount of selected features.

Further, in class, you discussed trying Forward/Backward Selection, but this ignores the fact that some variables may not be significant without the presence of other variables. The solution to this is Randomized Feature Selection.
