Overview¶

During this analysis, we will look for trends and seasonality, understand if the dataset exhibits stationarity, analyze ACF and PACF graphs, perform a seasonal decomposition of the data set, and build models to forecast what the thickness of the ice might be in the future.

We will do the forecasting using the following two models:

Dickey-Fuller Test¶

This test tells us how strongly a time series is defined by a trend by checking if there is a unit root. A unit root is a stochastic trend and the presence of one tells us that the pattern is unpredictable.

NULL Hypothesis: Suggests that the time series data has a unit root, meaning it is non-stationary.

Alternative Hypothesis: Suggests that the time series data does not have a unit root, meaning it is stationary.

We will reject the Null Hypothesis if we see a p-value lower the .05 (a 95% confidence level).

Sources:

https://machinelearningmastery.com/time-series-data-stationary-python/

https://www.statisticshowto.com/unit-root/

Autocorrelation¶

This tells us the autocorrelation between some point and another point at previous point in time. This includes direct and indirect dependencies.

Partial Autocorrelation¶

This also tells us the autocorrelation between some point and another point at a previous time. However, partial autocorrelation only includes direct dependencies (t-1->t)

AutoRegressive¶

According to the source below, if the PACF graph experiences a more sharp cutoff while the ACF graph experiences a more a gradual decay, we can say that the data displays an "AR signature". Further, if the first lag is a positve correlation this further indicates that we should add an AR term to the model. We should add as many AR terms that we see a significant correlation in the PACF graph.

We will add somewhere between 10 and 12 AR terms (we may try both).

This means that p=10 or p=12

Moving Average¶

Directly from the source below "If the ACF of the differenced series displays a sharp cutoff and/or the lag-1 autocorrelation is negative--i.e., if the series appears slightly "overdifferenced"--then consider adding an MA term to the model. The lag at which the ACF cuts off is the indicated number of MA terms."

I will not be adding a Moving average term as there is not a sharp cutoff or a negative lag-1 autocorrelation.

This means that q=0

ARIMA¶

What does p q d mean?¶

p = the number of autoregressive terms¶

p is the number of autoregressive terms (AR part). It allows to incorporate the effect of past values into our model. Intuitively, this would be similar to stating that it is likely to be warm tomorrow if it has been warm the past 3 days.

d = the number of nonseasonal differences¶

d is the number of nonseasonal differences needed for stationarity. Intuitively, this would be similar to stating that it is likely to be same temperature tomorrow if the difference in temperature in the last three days has been very small.

q = the number of moving-average terms¶

q is the number of lagged forecast errors in the prediction equation (MA part). This allows us to set the error of our model as a linear combination of the error values observed at previous time points in the past.

https://towardsdatascience.com/unboxing-arima-models-1dc09d2746f8

Selecting model parameters¶

p: 10 or 12

SARIMA¶

Whats wrong with ARIMA?¶

Before we go into SARIMA, we should point out what is wrong with ARIMA.

ARIMA is good for forecasting data, but it does not handle seasonal data well.

SARIMA¶

SARIMA is built off of ARIMA, and it supports seasonal data (such as the data set we are using here).

There are four new parameters:

  • P: Seasonal autoregressive order.
  • D: Seasonal difference order.
  • Q: Seasonal moving average order.
  • m: The number of time steps for a single seasonal period.

Selecting parameters¶

Per Duke University, a few rules are defined:

Differencing (d)¶

If the series has a strong and consistent seasonal pattern, then you should use an order of seasonal differencing--but never use more than one order of seasonal differencing or more than 2 orders of total differencing (seasonal+nonseasonal).

We have a strong and consistent season pattern, so we set differencing, d = 1.

AutoRegressive (p) and Moving Average (q)¶

If the autocorrelation at the seasonal period is positive, consider adding an SAR term to the model. If the autocorrelation at the seasonal period is negative, consider adding an SMA term to the model. Try to avoid mixing SAR and SMA terms in the same model, and avoid using more than one of either kind.

The autocorrelation at the seasonal period is positive, so we will add a SAR term to the model. We will try p = 1 and q = 0.

Periodicity¶

The number of periods in season. We have monthly data, so we will use 12. This can be verified in our ACF graph where the highest lag is at 1 and again at 12. This means that we will try s = 12.
